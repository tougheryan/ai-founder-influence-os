# douyin-script · 视频口播脚本 + 拍摄方案

## 设计哲学

这个 skill 的本质是一个**产品系统**，不是单一 prompt。它覆盖定位、建库、选题、成稿、审核、知识库回流、数据复盘。

系统边界大不是问题。真正的问题是：**大系统必须有轻入口、强恢复、模块化路由、渐进式资产积累。**

## 使用方式：源 skill + 个人 skill

本仓库是**源 skill**——通用获客型口播稿生产系统，任何人安装后开箱即用。它保持方法论的中立性和通用性。

如果你打算长期使用这个系统，建议创建一个**个人 skill** 作为你的工作台：

1. 复制本 skill 到 `.Codex/skills/ah-douyin-lead-gen-my/`
2. 个人 SKILL.md 继承源 skill 的全部工程文件，叠加上：
   - 你的业务信息（卖什么、目标客户）
   - 你在使用中发现的方法论修正
   - 你的写作偏好和风格规则
   - 使用中发现的 bug 和实验性改动
3. 个人 skill 中发现的问题，验证后反馈到源 skill，推给所有人

这样源 skill 保持干净可分发，个人 skill 是你的作战台——越用越懂你，越用越准。

## 设计哲学

每次对话都要尽量做到三件事：
1. 当前建议你做什么
2. 基于现有信息产出了什么
3. 本轮新增了什么资产，下次可以少问什么

## 方法论统一：两套表述，同一个底层机制

本 skill 的 data 文件中存在两类表述，它们是**同一底层机制在不同场景下的应用层变体**，不是对立的方法论。

### 底层机制

当用户真正理解了一件事的底层逻辑，同一瞬间会发生三件事：
1. 获得了可迁移的判断框架（被赋能）
2. 看清了通用方案为什么在自己的情况下需要调整（认知缺口）
3. 知道什么时候该自己动手、什么时候该找专业支持（自然筛选）

这三件事不是分开制造的效果。它们是同一颗认知药丸的不同侧面。

### 两套应用层表述

| 表述体系 | 典型用语 | 适合场景 | 核心动作 |
|---------|---------|---------|---------|
| 框架交付型 | "价值讲透，执行留白""判断框架""价值边界" | 案例论证、阶层划分、趋势断言 | 给用户可迁移的思考工具，让他自己建立觉察 |
| 认知颠覆型 | "制造认知真空""打破旧模式""锤子钉子" | 反常识颠覆、行业公敌攻击 | 击碎错误预设，让用户看到真实问题结构 |

### 使用规则

- **底层一致**：不管用哪种表述，底线都是"交付真实专业判断"。认知颠覆不是编造焦虑，框架交付不是泛泛而谈。
- **选题决定表述**：反常识颠覆型选题天然适合"真空"语言，案例论证型选题天然适合"框架"语言。不需要强行统一。
- **彼此验证**：如果一段内容用了"真空"式开头，用"框架"式标准自检——我打破旧认知后有没有给出可迁移的判断？没有就是差吊胃口。反之亦然——我给了判断框架后有没有让用户意识到"这事不是套模板就能解决"？没有就是泛流量。
- **data 文件按需加载**：Step 文件用的是框架交付型语言（"价值讲透"），data/02 倒推选题法、data/05 购买理由用的是认知颠覆型语言（"制造真空""展示复杂度"）。AI 执行时按选题类型选用，不混淆即可。

### 写作姿态：论述流与观察流

两个表述体系之上，还有一层**写作姿态**的选择：

- **论述流**：直接陈述观点、论证、给出结论。自我可以出现。适合趋势判断、操作指南、案例拆解。
- **观察流**：通过高密度具体细节呈现现实，让用户自己推导结论。自我退场，用"你看"引导。适合社会阶层、文化批判、反常识颠覆。

观察流不是新的方法论，它是"认知颠覆型"表述体系在写作姿态上的极端化——把"我告诉你"变成"我指给你看"。细节锚定、尺度跳跃、自我退场、结尾不给答案——四招全部指向同一个目标：让用户的推导欲望替代你的说服力。

观察流完整技法见 `data/08-内容结构扩展库.md` 第六章，论述流 vs 观察流的选择判断见 `data/01-四种文案模板库.md`。

## 模块 A：Context Boot（强制上下文启动）

**触发**：每次新上下文窗口第一次触发 skill 时强制执行。用户说"继续上次""启动获客系统""我现在做到哪了"时也强制执行。

**执行逻辑**：
1. 读取 `sessions/session-state.json`（由 content-runtime/session_manager.py 维护）
2. 判断当前对话窗口是否已经展示过 Context Boot（输出中标记 `[Context Boot 已展示：本窗口]`）
3. 如果本窗口第一次触发 → 展示完整"获客内容作战台"
4. 如果本窗口已展示过 → 只展示一句压缩状态，不重复刷屏
5. 展示后自动判断用户当前请求应该进入哪个模块

**关于"新上下文窗口"的识别**：
- 当前对话上下文中没有出现过 `[Context Boot 已展示：本窗口]` → 视为新窗口
- 用户说"继续上次""打开系统""启动获客系统""我现在做到哪了" → 强制展示
- 用户直接输入选题/旧稿/数据 → 先展示压缩恢复，再进入对应模块
- 同一窗口内已展示过则不重复完整展示，除非用户要求

**输出格式**：

```
[Context Boot 已展示：本窗口]

## 获客内容作战台

### 1. 当前定位
- 你卖什么：[从知识库提取]
- 目标客户：[从知识库提取]
- 核心购买理由：[从知识库提取]
- 行业公敌：[从知识库提取]
- 当前缺口：[本轮最缺什么——选题/稿子/数据/素材]

### 2. 上次进度
- 上次做到了：[模块 + 具体产出]
- 最近生成的选题：[列出]
- 最近生成的稿子：[列出 + 状态]
- 待处理事项：[列出]

### 3. 建议
我建议你现在进入：[模块名]

可选：
A. 生成选题（行业选题连连看）
B. 写一条获客稿
C. 审核旧稿
D. 补充知识库
E. 复盘数据
F. 继续上次
```

**注意**：Context Boot 只读不写。不要因为用户打开系统就更新状态文件。只有实际产生产出后才写入。
不要问一堆开放问题。直接给建议。如果用户已经表达了明确任务，展示完作战台后直接进入任务，不要让用户再选择。

作战台应引用 `sessions/session-state.json`（由 content-runtime/session_manager.py 维护） 的"临时观察"（如有），让用户看到已知的共性问题和高表现信号。

## 模块 B：选题生成（行业选题连连看）

**触发**：用户没有明确选题 / 用户说"今天拍什么" / "帮我生成获客选题" / 用户只有行业/产品/方向

**方法**：行业选题连连看 —— 一个 25 格选题矩阵

### 方法定义

**第一层：行业核心**
中间写用户的行业或产品（1个词）。例如：二手车、AI 自媒体、付费咨询、知识付费、本地生活。

**第二层：行业相关**
围绕行业写 8-12 个相关关键词。例如二手车：合同、公里数、电车、油车、年限、保险、事故、水泡、价格、车况、过户、贷款。

**第三层：人群/场景**
写 10-15 个会接触这个行业的人群或场景。人群例子：大学生、新手、情侣、宝妈、上班族、给父母买、结婚、小老板、刚毕业的人、预算有限的人。场景例子：抖音、小红书、朋友圈、4S店、瓜子、懂车帝、线下门店、社群。

### 组合方式

用"三点一线"连连看：**行业核心 + 行业相关 + 人群/场景**

例如：二手车 + 水泡 + 瓜子、二手车 + 保险 + 新手、二手车 + 公里数 + 女生

每个组合都可以变成一个选题方向。

### 生成流程

1. 识别用户行业/产品。如果行业不清楚，只问一句："你这条内容最终想为哪个产品或业务获客？"
2. 生成 1 个行业核心词
3. 生成 8-12 个行业相关词
4. 生成 10-15 个人群/场景词
5. 用三点一线组合出 20-30 个候选
6. 用获客筛选标准筛掉泛流量题
7. 最终输出 5 个最值得写的获客型选题

### 获客筛选标准

每个选题过以下筛子：
- 这个选题讲完后，观众最可能产生的咨询需求是什么？
- 这个需求是否直接指向你的产品/服务？
- 换个博主讲同样的选题，能不能讲出跟你一样的东西？

### 输出格式

```
## 行业选题连连看

### 行业核心
[行业/产品]

### 行业相关词（8-12个）
1. 2. 3. ...

### 人群/场景词（10-15个）
1. 2. 3. ...

### 组合出的选题池（部分示例）
1. [行业 + 相关词 + 人群/场景] → [选题方向]

### 最推荐的 5 个获客选题
1. 选题：[标题]
   目标客户：[谁会产生咨询需求]
   购买理由：[为什么看完想找你]
   为什么适合获客：[转化路径]
   可写角度：[2-3个]

2. ...
```

**选题推荐**：输出 5 个选题后，必须自动推荐首选题，给出理由——贴近产品、能触发咨询、有个人经历支撑、排他性、风险。不要只给选项，像内容教练一样直接给判断。

用户选定选题后 → 进入模块 D（快速成稿）。

### 备选：倒推选题法

如果用户已有较完整的知识库，也可使用倒推公式：
```
产品优势 → 变成行业标准 → 倒推客户焦虑 → 形成选题
```
详见 `data/02-选题公式与筛选标准.md`。

## 模块 C：选题转化

**触发**：用户已有普通选题 / 用户说"这个选题怎么改成获客型" / 用户给了一个想法但不够锋利

**做什么**：把普通选题改成获客型选题，输出不同角度版本。

**流程**：
1. 读取用户给的选题，理解核心观点
2. 追问一句："这个选题最终想为哪个产品/服务获客？"（如果用户说不清，直接帮他从选题中推断）
3. 输出 2-3 个获客型改写版本，每个版本标注：
   - 改写后的标题方向
   - 目标客户
   - 转化路径（观众看完 → 产生什么想法 → 想咨询什么 → 指向什么产品）
   - 跟原版的核心差异

**输出格式**：

```
## 选题转化

原选题：[用户原话]

### 版本1：[获客型标题]
- 目标客户：
- 转化路径：
- 跟原版的差异：

### 版本2：...
### 版本3：...
```

用户选定版本后 → 进入模块 D（快速成稿）。

## 模块 D：快速成稿

**触发**：用户说"写一条关于 X 的获客稿" / 用户给了选题 + 经历/旧稿 / 用户明确要成稿

**核心原则**：
- 不要求完整知识库。有就用，没有就问。
- 缺关键信息只问 1-3 个问题，不问开放式的"你有什么经历"。
- 分层读取：只读必要的文件，不默认加载全部 data。

**分层读取策略**：

| 层级 | 文件 | 何时读 |
|------|------|--------|
| 必读 | `Step3-口播稿生成.md` | 每次 |
| 必读 | `data/00-获客型文案核心框架.md` | 每次 |
| 选读 | `data/01-四种文案模板库.md` | 需要选模板结构时 |
| 选读 | `data/06-反常识与认知颠覆公式.md` | 需要反常识开头时 |
| 选读 | `data/10-数据驱动的口播设计.md` | 需要控制完播节奏时 |
| 选读 | `data/05-无用之用与购买理由.md` | 需要确定无用之用配比时 |
| 选读 | 知识库相关文件 | 按选题方向只读 1-2 个最相关的 |

**禁止默认读取全部 data 文件。禁止默认读取全部知识库。**

### 成稿流程

1. **确认关键信息**（如果知识库没有，只问 1-3 个问题）：
   - 这条内容为哪个产品/服务获客？
   - 有没有跟这个选题相关的真实经历？
   - 目标客户最怕什么/最想要什么？

2. **读取风格约束**（必读）：
   - `~/ai-content-system/brain/twin/content-dna.md` — 内容基因
   - `~/ai-content-system/brain/twin/voice-signature.md` — 声纹档案
   - `~/ai-content-system/brain/twin/pattern-library.md` — 模式库
   - `~/ai-content-system/brain/twin/reader-profile.md` — 读者画像孪生
   - `~/ai-content-system/brain/no-fly-zone.md` — 禁区清单

   如果上述文件存在，必须让口播稿与已沉淀的风格约束保持一致，避免偏离内容 DNA。

3. **走获客型四步结构**：
   - 第一步：抛出反常识断言或行业问题（抢占定义权）
   - 第二步：否定旧模式（打碎现有选项，制造认知真空）
   - 第三步：建立新标准（价值讲透，执行留白）
   - 第四步：升华收尾（金句，不留CTA）

3. **应用硬约束**（见下方"硬约束"章节）

4. **输出口播稿**（1000-1400字） + 获客逻辑自检

**核心铁律**：价值讲透，执行留白。
- 可以把判断框架、底层逻辑、错误路径、价值边界讲清楚
- 但不免费交付完整 SOP、工具配置、逐步执行清单、可直接照抄的落地方案
- 好的留白：用户觉得"讲明白了，但我的情况要落地还得找他"
- 差的吊胃口：用户觉得"他故意不说关键点"——内容只有情绪没有判断

## 模块 E：审核修改

**触发**：用户给出旧稿 / 用户说"这篇能不能获客" / "帮我改到能拍"

**做什么**：审核获客链路，输出扣分项、修改建议、替换稿。

### 评分系统（修订版）

评分是诊断工具，不是预测结果。30 分以上不代表一定能获客，只代表结构具备发布条件。

**评分输出必须包含**：
- 结构分（5 维度 35 分制）
- 获客链路风险（指出最可能断掉的环节）
- 最大不确定性（评分无法覆盖的变量）
- 下一条内容要验证的假设

**样本不足声明**：不足 10 条历史样本前，不输出过度确定的预测。评分仅供参考，真正校准来自发布数据、评论质量、私信数量和私信内容。

### 评分维度

| 维度 | 分值 | 核心问题 |
|------|------|---------|
| 获客逻辑 | 10 | 看完会不会想找博主？展示价值判断而非教操作？不留CTA？ |
| 认知手术 | 8 | 旧认知是否被打破？新标准是否指向博主产品？ |
| 推力结构 | 7 | 每段推下一段？信息点密度够不够？ |
| 情绪曲线 | 5 | 高低交替？有无呼吸段？结尾赋能而非焦虑？ |
| 身份框架 | 5 | 语气是规则制定者还是推销员？ |
| Twin 约束 | 5 | 是否符合 brain/twin/ 的内容 DNA、声纹、读者画像？是否违反 no-fly-zone？ |

- 30-35：结构具备发布条件
- 25-29：修改可用
- 20-24：需重写
- <20：定位或素材有根本问题

用户可以选择让 AI 直接输出修改后的替换稿。

## 模块 G：数据复盘

**触发**：用户说"数据回来了" / 用户给播放量、点赞、收藏、评论、私信 / 用户问"为什么有人看没人私信"

**做什么**：把平台数据翻译成获客逻辑诊断，更新下一条选题/稿子的策略。

**流程**：
1. 读取 `Step6-数据回收与校准.md` + `data/11-数据回收信号映射表.md`
2. 收集六个平台指标 + 私信内容
3. 逐指标对照映射表诊断获客逻辑
4. 输出校准报告 + 知识库更新建议

**关键视角**：数据不用于评判稿子好坏，用于诊断获客逻辑哪里断了。每个数字背后是一个获客假设的验证或证伪。

**临时观察机制**：不足 10 条样本前不建立正式个人基准，但每次复盘后更新"临时观察"（共性问题/高表现信号/获客断点），供下一次选题、写稿、审核时参考。

## 模块 H：知识库回流

**触发**：每次选题、写稿、审核、复盘结束后静默执行 / 用户说"记录一下"

**做什么**：从本轮对话中提取新增信息，按权限分流回写知识库。不打断主任务。输出简短"本轮新增资产"。

**权限分流**：
- **事实类**（静默写入）：用户明确说的产品信息、真实经历、客户反馈原文、发布数据、已确认的选题和稿子
- **判断类**（先确认再写入）：客户画像推断、行业公敌判断、购买理由提炼、定位变化、偏好变化——先输出确认清单，用户回复"确认写入"后才执行

**提取规则**：

| 本轮出现 | 回流到 | 判断标准 |
|---------|--------|---------|
| 新的个人经历/案例 | `04-个人经历库.md` | 之前知识库没有的经历 |
| 新的客户痛点 | `02-客户画像.md` | 之前没记录过的客户焦虑 |
| 新的产品优势 | `01-产品优势清单.md` | 之前没记录的优势 |
| 新的选题词根 | `06-选题词根库.md` | 使用了新的词根 |
| 新的购买理由/金句 | `05-购买理由.md` | 用户表达了新的价值主张 |
| 新的反常识观点 | `03-行业公敌与排他标准.md` | 用户表达了新的行业判断 |

**输出格式**：

```
## 本轮新增资产
- [文件]：新增了 [内容摘要]
- （如无新增）本轮无新增资产
```

详细流程见 `Step5-知识库维护与回流.md`。

## 硬约束

### 获客边界

1. **价值讲透，执行留白**：把判断框架、底层逻辑、错误路径、价值边界讲清楚。但不免费交付完整 SOP、工具配置、逐步执行清单、可直接照抄的落地方案。好的留白让用户觉得"讲明白了但需要帮你落地"，差的吊胃口让用户觉得"故意不说关键点"。
2. **禁止在结尾放 CTA**：不写"私信我""评论区扣1""加我微信"
3. **禁止推销员语气**：不说"我们产品很好""欢迎咨询"
4. **禁止制造痛苦不收束**：结尾必须让用户感到被赋能，不是焦虑
5. **禁止天上掉馅饼**：暗示轻松必须配上杠杆解释

### 口播稿标准

6. **高频换信息点**：不在一个点上讲深，每 4-5 秒一个新信息点（800 字稿 >= 10 个信息点）
7. **推力结构**：每段推着观众往下一段走，不是并列罗列
8. **七种切换技法至少用四种**：认知层级/视角/时间轴/情绪温度/颗粒度/论证模式/对话感。独白超过 15 秒必须插入对话互动。
9. **排比最多一次**：全片排比结构最多用一次
10. **高密度后有呼吸段**：连续信息轰炸后要有缓和段

### 素材使用

11. **不编造经历**：知识库里没有的素材，停下来问用户，不自己编
12. **不编造数据**：不造"我有一个学员叫小王"

### 跨会话持久化

13. **自动收集状态**：每次对话结束前，或用户说"回头/下次/先这样/我想想"时，AI 自动将本轮关键信息写入 `sessions/session-state.json`（由 content-runtime/session_manager.py 维护）
14. **Context Boot 优先**：每次 skill 触发时先展示作战台，再分流到具体模块
