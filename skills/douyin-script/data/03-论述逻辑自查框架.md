# 论述逻辑自查框架

> 来源：获客型文案白皮书 第八章

写完稿子后，用这套框架逐条检查论述逻辑是否成立。6项检查，每条必须通过。

---

## 一、推力结构检查

- [ ] 每一段是否在推着观众往下一段走？（不是并列罗列）
- [ ] 相邻段落之间是否存在"为什么会这样？→ 因为..."的逻辑链条？
- [ ] 有没有哪一段删掉后全文逻辑不受影响？（如果有，那段就是无效的）

**推力结构三种实现方式**：
1. **悬念递进**：每段结尾提前"剧透"下一段的hook，但不说完
2. **信息密度控制**：每15秒必须有一个新信息点投入
3. **情绪波动**：高→低→高→低→高，不能连续两段以上情绪持平

---

## 二、认知手术检查

- [ ] 有没有一个明确的"旧认知"被打破了？
- [ ] 有没有一个明确的"新标准"被建立了？
- [ ] 新标准和你的产品/服务之间是否存在必然联系？
- [ ] 打破旧认知的方式是"指出行业信息差/揭露大家都在踩的坑"而非"攻击某类人"？

---

## 三、身份框架检查

- [ ] 你的语气是"规则制定者"（教父）还是"推销员"？
- [ ] 你有没有在"告诉用户世界是什么样的"而非"求着用户买"？
- [ ] 有没有出现"我们产品很好""欢迎咨询""私信我"等低位话术？

**高位 vs 低位的判断标尺**：

| 低位（推销员） | 高位（教父） |
|--------------|------------|
| 求着客户买 | 告诉客户世界的规则是什么 |
| "我们的产品很好" | "这个行业的本质是这样的" |
| 让客户做裁判 | 你来做裁判，客户是学生 |
| "你可以试试" | "你现在有理解了吗" |

---

## 四、反向筛选检查

- [ ] 看完这条视频，"劣质客户"会不会觉得你讲得太虚、没给具体方法而划走？
- [ ] "优质客户"会不会产生"这个人说的太对了"的共鸣？
- [ ] 如果两者都满足，反向筛选生效。

---

## 五、网球拍检查（制造美好 vs 制造痛苦）

- [ ] 你是在制造痛苦（告诉用户他哪里不行）还是在制造美好（告诉用户他可以更好）？
- [ ] 用户看完后的情绪是焦虑还是被赋能？
- [ ] 如果是焦虑——他大概率不会找你买单。需要调整。
- [ ] 如果是被赋能——他大概率会想找你帮他实现。方向正确。

**核心公式**：
- 讲干货 = 给用户施加压力 → 他在你这感到痛苦 → 不会买单
- 讲无用 = 给用户提供情绪的避风港 → 他在你这感到美好 → 主动买单

---

## 六、杠杆包装检查

- [ ] 如果你的文案暗示了"轻松/省力"，有没有同时给出合理的杠杆解释？
- [ ] 是"动动手指就能赚钱"（低级，吸引白嫖党）
- [ ] 还是"用系统代替人力"（高级，吸引有付费能力的认知人群）

**铁律**："轻松"必须披上"重杠杆"的外衣。不能让用户觉得是天上掉馅饼，要让他觉得是他认知升级了。

---

## 不同客单价的策略配比

| 客单价 | "有用/无用"配比 | 内容策略 |
|--------|---------------|---------|
| 低价（<1000） | 7分有用/3分无用 | 可以偏干货，给即时获得感 |
| 中价（1000-10000） | 5分有用/5分无用 | 干货建立信任 + 无用拉升渴望 |
| 高价（>10000） | 2分有用/8分无用 | 几乎不讲方法，只讲认知和身份 |
