# 数据回收信号映射表

> 平台数据指标 → 获客逻辑诊断 → 知识库更新动作
> 本文件是 Phase 6 的核心翻译层

---

## 一、映射逻辑

数据不是用来"看"的，是用来**诊断获客逻辑哪里出了问题**的。每个数据指标对应一个获客假设的验证。

```
数据信号 → 获客诊断 → 知识库更新动作
```

---

## 二、指标映射表

### 2秒跳出率

| 数据信号 | 获客诊断 | 知识库更新动作 |
|---------|---------|--------------|
| <30%（优秀） | 认知炸弹生效，钩子击中了用户预设 | 记录本条使用的钩子类型为偏好（更新选题偏好） |
| 30-50%（正常） | 钩子及格，但不突出 | 不更新 |
| >50%（差） | 钩子太弱或太慢，前2秒在铺垫/自我介绍/寒暄 | **写作偏好**：后续稿子强制第一句用 data/06 的反常识公式；**选题偏好**：减少"行业定义型"公式，增加"反常识否定型" |

### 5秒完播率

| 数据信号 | 获客诊断 | 知识库更新动作 |
|---------|---------|--------------|
| >60%（优秀） | 前5秒信息点密度够，预期管理做得好 | 记录本条的信息点投放节奏为偏好 |
| 50-60%（及格） | 维持住了但没惊喜 | 不更新 |
| <50%（差） | 前5秒信息点太少，或没给"我接下来要讲什么"的预期 | **写作偏好**：后续稿子前5秒强制塞 3 个信息点 |

### 整体完播率

| 数据信号 | 获客诊断 | 知识库更新动作 |
|---------|---------|--------------|
| >40%（优秀） | 信息点切换频率 + 论证模式交替 + 对话感都在起作用 | 记录本条的结构策略为偏好 |
| 20-40%（正常） | 一般，中等水平 | 不更新 |
| <20%（差） | 某一段出现连续同模式导致流失 | **写作偏好**：回到 Phase 4 审核报告，找到情绪持平或同论证方式的段落，标记该模式为"个人雷区" |

### 点赞率

| 数据信号 | 获客诊断 | 知识库更新动作 |
|---------|---------|--------------|
| >5%（优质） | 共情触发器生效——反常识命名/反问直击经验/身体细节唤起记忆 | 记录使用的触发方式类型为偏好 |
| 3-5%（正常） | 及格 | 不更新 |
| <3%（差） | 内容太抽象，缺少"说出他心里话"的细节 | **写作偏好**：后续稿子强制每 300 字埋一个"反常识命名"或一个"反问直击经验" |

### 收藏率

| 数据信号 | 获客诊断 | 知识库更新动作 |
|---------|---------|--------------|
| >5%（高） | 框架性内容起效，用户觉得"以后用得上" | 记录"埋框架钩子"策略有效，保持 |
| <3%（低） | 纯观点输出，没有框架钩子 | **写作偏好**：后续稿子强制在中间段埋一个框架提示（"这其实是一个X步的判断框架"） |

### 评论率

| 数据信号 | 获客诊断 | 知识库更新动作 |
|---------|---------|--------------|
| 评论多且讨论方向对 | 争议口或身份带入起效 | 记录使用的评论触发方式为偏好 |
| 评论多但全是骂 | 争议口太猛，攻击了人而非模式 | **写作偏好**：降低"树敌帖"比例，增加"造梦帖" |
| 评论少 | 没留讨论空间，结尾太封闭 | **写作偏好**：后续稿子结尾加一个"不给答案的开放问题" |

### 私信内容（最关键）

| 数据信号 | 获客诊断 | 知识库更新动作 |
|---------|---------|--------------|
| 私信问"怎么做""能不能教" | 获客逻辑正确：价值讲透成功，执行留白成功，用户认可判断但需要落地帮助 | **客户画像**：记录私信者的身份特征和语言习惯 |
| 私信问"你是做什么的""你卖什么" | 获客逻辑半成功：人设建立了但购买理由不够清晰 | **购买理由**：后续稿子加强金句和关键词的植入密度 |
| 私信问具体技术细节 | 获客逻辑失败：执行留白不够，暴露了太多可复制的操作细节，用户在验证而非求助 | **写作偏好**：严重警告——下一条稿子必须收紧执行留白 |
| 没有私信 | 整个获客链路断了 | 检查定位声明和行业公敌是否正确 |
| 私信者是劣质客户 | 反向筛选失效 | **客户画像**：检查"劣质客户"的画像是否需要收紧 |

## 三、基准值校准规则

以下基准值在用户积累 ≥10 条稿子的数据后可以个性化调整：

| 指标 | 通用基准 | 个性化校准方法 |
|------|---------|--------------|
| 2秒跳出率 | <30%优 / 30-50%常 / >50%差 | 取用户历史稿子的中位数作为个人基准 |
| 5秒完播率 | >60%优 / 50-60%常 / <50%差 | 同上 |
| 整体完播率 | >40%优 / 20-40%常 / <20%差 | 同上 |
| 点赞率 | >5%优 / 3-5%常 / <3%差 | 同上 |

当稿子追踪表中"已回收"的条目 ≥10 条时，Phase 6 自动计算个人基准并用它替代通用基准。

## 四、信号组合诊断

单个指标看的是局部问题，多个指标组合看的是系统性偏差：

| 组合信号 | 诊断 | 根因 |
|---------|------|------|
| 2秒跳出高 + 5秒完播低 | 开头全线崩溃 | 钩子不行，选题方向可能就有问题 |
| 5秒完播低 + 整体完播正常 | 开头弱但正文强 | 钩子需要换，正文结构保留 |
| 整体完播低 + 收藏高 | 内容好但节奏差 | 信息点有价值但切换频率太低 |
| 点赞低 + 收藏高 | 太理性，缺情绪 | 加情绪温度切换和共情触发器 |
| 完播高 + 点赞高 + 无私信 | 内容是好的，但没获客 | 获客逻辑断了——可能价值没讲透（用户觉得"知道了"就划走），或执行留白没形成（用户觉得能自学，不需要找你） |
| 私信多但全是劣质客户 | 反向筛选失效 | 回 Phase 1 检查客户画像的"劣质客户"定义 |
