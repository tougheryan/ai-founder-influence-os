# content-intake · 素材理解

## 作用

把原始素材变成结构化的内容原料。不是简单摘要，而是提取后续角度生成和内容拆解能直接使用的要素。

## 触发方式

- `/content-intake`
- `理解素材`、`分析素材`、`拆解一下这个素材`
- `这个人/这件事值得写什么`

## 输入

任意形式的素材：
- 文字 / Markdown
- PDF / DOCX / 网页链接
- 语音转文字
- 图片（配 OCR 或人工描述）
- 聊天记录 / 会议纪要
- 用户口头描述

可选参数：
- `scenario`：素材场景类型，如未指定需主动询问或反推
- `platforms`：目标平台（wechat / xhs / douyin），默认全部
- `research`：是否需要补充研究，默认 false

## 素材场景分类

收到素材后，先判断属于哪一类场景。场景决定后续的内容结构、标题逻辑和平台表达方式。如果用户没有明确说明，根据素材反推并确认；如果跨多个场景，以**主场景**为准，其他作为补充。

| 场景 | 核心用途 | 适合内容 | 主要平台 |
|---|---|---|---|
| **个人履历 / 经验 / 知识分享** | 建立专业身份和可信背书 | 职业经历、方法论、行业判断、复盘 | 视频号、小红书、公众号 |
| **访谈 / 嘉宾出席 / 演讲分享** | 将线下高价值对话沉淀为内容资产 | 活动纪实、观点金句、人物背书 | 视频号、公众号、Twitter/X |
| **个人成长 / 人生心得** | 增强真实感和情绪连接 | 转折、选择、压力、修复、长期主义 | 小红书、视频号 |
| **日常工作 / 生活 Vlog** | 将碎片化日常变成稳定素材池 | 出差、会议、客户拜访、工作现场 | 小红书、TikTok、视频号 |
| **业务 / 产品增长 / 引流获客** | 把内容转化为商业信任入口 | 客户案例、产品价值、服务说明、转化 CTA | 公众号、视频号、Twitter/X |
| **全球化表达 / 跨境内容** | 面向海外用户、东南亚华人和关注中国市场的专业人士 | 双语内容适配 | TikTok、Twitter/X、小红书 |

## 输出

保存到：

```
~/ai-content-system/pipeline/intake/{YYYY-MM-DD}-{slug}-intake.md
```

**必须通过 content-runtime 写入**，以便后续步骤自动追踪最新文件：

```bash
python $HOME/plugins/ai-founder-influence-os/skills/content-runtime/scripts/content_runtime.py \
  --project ~/ai-content-system \
  write --step intake --slug {slug} \
  --content @/path/to/intake-body.md \
  --source "用户输入"
```

如果不希望从文件读入内容，也可以把 `--content` 后的 `@/path/to/intake-body.md` 换成用引号包裹的 markdown 字符串。

输出结构：

```markdown

## 1. 人物
- 核心人物：...
- 人物关系：...
- 人物标签：...

## 2. 场景
- 时间 / 地点 / 背景
- 关键事件链

## 3. 观点
- 明确观点：...
- 隐含立场：...
- 可争议点：...

## 4. 情绪
- 主情绪：...
- 情绪曲线：...
- 可共鸣点：...

## 5. 案例
- 核心案例：...
- 支撑细节：...
- 可展开度：⭐⭐⭐⭐⭐

## 6. 商业意图
- 产品 / 服务 / 品牌信息
- 目标受众

## 7. 不可公开信息
- 隐私 / 商业机密 / 敏感关系
```

## 工作流

1. **读取素材**：根据输入类型选择读取方式
2. **判断场景**：识别素材属于哪一类素材场景，如未明确则主动询问或反推，以主场景为准
3. **提取要素**：按 7 个维度识别人物、场景、观点、情绪、案例、商业意图、不可公开信息
4. **补充研究**（可选）：调用 `aihot`/`hv-analysis`/`deep-research` 补充行业背景
5. **输出报告**：调用 content-runtime 写入 `pipeline/intake/`，并给出下一步建议

   示例：
   ```bash
   python $HOME/plugins/ai-founder-influence-os/skills/content-runtime/scripts/content_runtime.py \
     --project ~/ai-content-system \
     write --step intake --slug {slug} \
     --content @/tmp/intake-body.md \
     --source "用户输入"
   ```

## 场景化提取提示

不同场景的关注重点应有所倾斜：

- **个人履历 / 经验 / 知识分享**：重点提取可证明专业度的经历、方法论步骤、关键判断依据、可复用框架
- **访谈 / 嘉宾出席 / 演讲分享**：重点提取金句、对话冲突、人物背景、现场细节、观点交锋
- **个人成长 / 人生心得**：重点提取转折点、情绪曲线、具体困境、修复过程、价值观变化
- **日常工作 / 生活 Vlog**：重点提取真实场景、可视化细节、节奏变化、可被共鸣的日常冲突
- **业务 / 产品增长 / 引流获客**：重点提取产品价值、用户痛点、转化路径、客户案例、CTA 入口
- **全球化表达 / 跨境内容**：重点提取跨文化差异、海外受众关注点、双语表达难点、区域市场特征

## 质量原则

- 不编造素材中没有的信息
- 不可公开信息必须显式标出
- 情绪判断要有文本依据
- 商业意图要区分"显式"和"隐式"
- 每个观点要标注来源位置

## 输入/输出示例

### 示例 1：语音转文字素材 → 素材理解报告

**输入**
```text
（语音转文字片段）
上周我连夜用 Codex 改一个老项目，本来预计要三天的活，结果一晚上就搞定了。我当时特别兴奋，但又有点慌，因为以前总觉得写代码是护城河，现在感觉这护城河可能没我想象的那么宽。我录这个是想聊聊我对 AI 编程的真实感受。
```

**预期输出**
```markdown
# 素材理解报告 · AI 编程助手的真实冲击

## 0. 素材场景
个人成长 / 行业观察（混合）

## 1. 人物
- 说话人：有 3 年左右开发经验的技术从业者
- 身份：创业者或技术负责人
- 情绪：兴奋 + 焦虑

## 3. 观点
- Codex 显著提升了编码效率（三天活一晚上完成）
- 写代码作为职业护城河的判断需要重新评估

## 6. 商业意图
通过真实体验建立专业信任，吸引对 AI 编程焦虑/好奇的读者
```
